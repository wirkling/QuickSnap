# QuickSnap Metadata Plugin for Claude Code

A Claude Code plugin that automatically surfaces AI-generated image descriptions when reading PNG/JPG files.

## What it does

When you use Claude Code to read an image file, this plugin automatically extracts any embedded description metadata (EXIF/TIFF) and injects it into the conversation context. This means Claude "understands" what's in the screenshot without relying solely on vision.

Works with:
- Screenshots from [QuickSnap](https://github.com/wirkling/QuickSnap) (which embeds AI-generated descriptions)
- Any image with `kMDItemDescription` or `kMDItemComment` metadata

## Requirements

- macOS (uses `mdls` for metadata extraction)
- Claude Code CLI

## Installation

```bash
claude plugin add /path/to/claude-plugin
```

Or clone the repo and point to the `claude-plugin` directory:

```bash
git clone https://github.com/wirkling/QuickSnap.git
claude plugin add ./QuickSnap/claude-plugin
```

## Components

### PreToolUse Hook (automatic)
Fires whenever Claude Code's `Read` tool targets a PNG/JPG file. If the image has embedded description metadata, it's injected as additional context automatically.

### `/read-metadata` command
Explicitly extract and display metadata from any image:

```
/read-metadata ~/Desktop/screenshot.png
```

## How QuickSnap writes metadata

QuickSnap uses `CGImageDestination` to embed AI-generated descriptions into the TIFF metadata of PNG files. The description is generated by Claude API based on the screenshot content, producing a concise summary of what's visible in the image.
